# Code Of Conduct

CCP is intended to be a constructive, vendor-neutral space for companion-animal context interoperability.

Participation in this project should improve the specification, examples, conformance tests, implementation guidance, and safety model.

## Expected Behavior

Participants are expected to:

- Be respectful and professional.
- Focus on technical substance.
- Assume privacy, consent, and safety concerns are legitimate until evaluated.
- Disclose relevant conflicts of interest.
- Keep vendor-specific advocacy separate from protocol design.
- Use synthetic data in public discussions and contributions.
- Accept maintainer decisions about scope, tone, and project direction.

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, threats, insults, or personal attacks.
- Publishing or requesting private customer, owner, caregiver, staff, facility, pet, billing, medical, or household data.
- Attempting to bypass consent, privacy, or security boundaries in examples or implementations.
- Repeatedly derailing design discussions after maintainers have made a scope decision.
- Misrepresenting draft CCP compatibility or claiming certification that the project has not granted.
- Using the project primarily to promote a vendor, product, or private implementation.

## Scope

This policy applies to project repositories, issues, pull requests, discussions, documentation, community channels, events, and private maintainer communications related to CCP.

## Enforcement

Maintainers may take action to preserve a productive and safe project environment, including:

- Asking participants to revise comments or contributions.
- Removing comments, issues, pull requests, or other contributions.
- Limiting participation in project spaces.
- Declining contributions that create privacy, safety, security, legal, or governance risk.

Maintainers should apply this policy consistently and explain moderation decisions when doing so would not create additional risk.

## Reporting

Report conduct concerns privately to the maintainers. Do not include private customer, owner, caregiver, staff, facility, pet, billing, medical, or household data in reports unless absolutely necessary, and avoid public disclosure of sensitive information.
