# Maintainers

CCP is maintained through this repository.

## Public Questions And Feedback

Use GitHub issues for:

- Commerce Context Profile feedback.
- Spec and schema change proposals.
- Conformance or example bugs.
- Adapter questions.
- Package helper feedback.

Use the design-partner feedback issue template when sharing implementation review notes.

## Security And Sensitive Reports

Do not open public issues for suspected vulnerabilities, consent-bypass behavior, privacy leaks, secrets, real customer data, real pet medical data, billing data, or sensitive facility data.

Use `SECURITY.md` for private reporting guidance.

## Maintainer Responsibilities

Maintainers should:

- Triage new issues with `needs-triage`.
- Keep draft status clear until compatibility is stable.
- Require conformance updates for normative behavior changes.
- Preserve vendor-neutral examples and language.
- Keep security-sensitive reports out of public issues.
- Record accepted draft changes in `CHANGELOG.md`.

## Current Maintainer Path

Until a dedicated maintainer email is published, use GitHub issues for non-sensitive project feedback and GitHub private vulnerability reporting for sensitive security or privacy reports when enabled.

## Public Source URL

The public source repository URL is not assigned yet. Do not publish package or adapter metadata with a repository URL until that URL resolves for unauthenticated users.
